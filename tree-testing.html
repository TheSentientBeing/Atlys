Evaluation Method

Tree Testing

Evaluate how easily users can find information in your website's menu structure by removing all styling, branding, and visual elements.

Time Required

1-2 Weeks

Participants Needed

30-50 Users

Difficulty Level

Intermediate

Overview

Tree testing evaluates the key aspects of menu structure and information architecture by presenting users with a stripped down version of the site's navigation hierarchy. This method helps validate content location and reveal structural problems users have with mental models and expectations.

Usage

When to use

  • Validating site structure
  • Testing navigation design
  • Restructuring content
  • Planning new sections
  • Improving findability
  • Before visual design
  • After card sorting
  • Evaluating terminology

When not to use

  • Testing visual design
  • Evaluating interactions
  • When content is undefined
  • Testing page layouts
  • Assessing aesthetics
  • For dynamic content
  • When navigation is simple

Process

1

Test Planning

  • Define test objectives
  • Create tree structure
  • Select tasks
  • Set success criteria
  • Choose testing tool
  • Plan analysis approach
2

Test Preparation

  • Input tree structure
  • Write task scenarios
  • Set correct paths
  • Test terminology
  • Pilot with colleagues
  • Refine tasks
3

Testing

  • Recruit participants
  • Launch study
  • Monitor responses
  • Track completion
  • Gather feedback
  • Note problems
4

Analysis

  • Calculate success rates
  • Review path choices
  • Identify patterns
  • Map common errors
  • Compare user paths
  • Assess task times
5

Reporting

  • Document findings
  • Visualise paths
  • Highlight issues
  • Make recommendations
  • Present alternatives
  • Plan improvements

Example Questions

  • "Where would you find information about X?"
  • "Under which category would you expect to find Y?"
  • "How confident were you in your choice?"
  • "What made this task difficult?"
  • "What would you call this section?"

Pairs well with

Card Sorting

Why?

Use tree testing to validate the information architecture that emerged from card sorting exercises.

Why?
