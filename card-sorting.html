Generative Method

Card Sorting

Observe how users categorize information during a card sorting session to gain insights into their mental models and preferences.

Time Required

2-3 Weeks

Participants Needed

15-20 Users

Difficulty Level

Intermediate

Overview

Card sorting is a research method where participants organise topics into categories that make sense to them. It helps researchers understand users' mental models and expectations, making it invaluable for designing intuitive information architecture and navigation structures.

Usage

When to use

  • Designing new navigation structures
  • Reorganizing existing content
  • Understanding users' mental models
  • Validating information architecture
  • Finding menu structures and taxonomies
  • Planning content hierarchies

When not to use

  • Testing interaction patterns
  • Evaluating visual design
  • Measuring performance metrics
  • When content is highly specialized or technical
  • When categories are already firmly established
  • For small amounts of content

Process

1

Study Planning

  • Define study objectives
  • Choose between open, closed, or hybrid sorting
  • Select content for cards
  • Determine if remote or in-person
  • Create study protocol
  • Set up card sorting tools
2

Card Preparation

  • Write clear card labels
  • Review content coverage
  • Create consistent formatting
  • Prepare category cards
  • Test cards with colleagues
  • Consider translations if needed
3

Session Preparation

  • Welcome and brief participants
  • Explain sorting process
  • Observe sorting behavior
  • Take notes on participant comments
  • Ask clarifying questions
  • Document category names and groupings
4

Data Collection

  • Record all category structures
  • Document participant rationale
  • Capture sorting patterns
  • Note problematic items
  • Track completion time
  • Save all sorting arrangements
5

Analysis & Synthesis

  • Analyze sorting patterns
  • Identify common groupings
  • Note outliers and conflicts
  • Look for content relationships
  • Create dendrograms if needed
  • Document findings and recommendations

Example Questions

  • "How would you group these items in a way that makes sense to you?"
  • "What would you name this group of items?"
  • "Could you explain why you grouped these items together?"
  • "What made you place this item in this category?"
  • "Are there any items you think are missing?"

Pairs well with

Tree Testing

Why?

Use tree testing to validate the navigation structure that emerges from card sorting, ensuring it works for users to complete real tasks.

Content Audit

Why?

Combine with content audit to ensure all relevant content is included in the card sort and to identify new content needed.
