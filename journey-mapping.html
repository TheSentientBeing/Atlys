Discovery Method

Journey Mapping

Create a comprehensive visualization of how users interact with your product across different channels and timeframes. Perfect for identifying pain points and opportunities.

Time Required

2-4 Weeks

Participants Needed

8-12 Users

Difficulty Level

Intermediate

Overview

Journey mapping creates a visual representation of users' experiences across touch points and time. This method helps teams understand the complete user experience, revealing pain points, emotional highs and lows, and opportunities for improvement, while building empathy across the team.

Usage

When to use

  • Understanding cross-channel experiences
  • Identifying pain points
  • Planning service improvements
  • Discovering opportunities
  • Breaking down silos
  • Prioritising improvements
  • Communicating user experience

When not to use

  • For simple, single interaction flows
  • When lacking user research data
  • Testing specific features
  • Measuring performance
  • For technical documentation
  • When immediate solutions are needed
  • Without stakeholder buy-in

Process

1

Research & Preparation

  • Gather existing research
  • Conduct user interviews
  • Collect analytics data
  • Define journey scope
  • Identify key personas
  • Set mapping objectives
2

Framework Creation

  • Define journey phases
  • Identify touch-points
  • Set map dimensions
  • Create timeline
  • Choose mapping tools
  • Design template
3

Mapping Workshop

  • Assemble stakeholders
  • Map current journey
  • Document pain points
  • Note opportunities
  • Capture emotions
  • Add supporting data
4

Detail & Enhancement

  • Add user quotes
  • Include metrics
  • Layer emotions
  • Map channels
  • Document systems
  • Note ownership
5

Validation

  • Review with users
  • Check with stakeholders
  • Validate touch-points
  • Confirm pain points
  • Test assumptions
  • Update findings
6

Activation

  • Present journey map
  • Prioritise opportunities
  • Plan improvements
  • Assign ownership
  • Set timelines
  • Track progress

Example Questions

  • "What are the key phases of the user journey?"
  • "Where do users experience friction?"
  • "How do emotions change throughout?"
  • "Which touch-points are most critical?"
  • "What opportunities exist for improvement?"
