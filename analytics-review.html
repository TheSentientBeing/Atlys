Behavioural Method

Analytics Review

Examine user behavior data from digital analytics tools to identify patterns, trends, and potential issues. Great for data-driven decisions.

Time Required

1-2 Weeks

Participants Needed

N/A (uses existing data)

Difficulty Level

Intermediate

Overview

Analytics review is a systematic examination of quantitative data collected from digital platforms to understand user behavior, identify trends, and uncover opportunities for improvement. This method leverages existing user interaction data to make data-driven decisions about design and functionality.

Usage

When to use

  • When you need quantitative evidence to support decisions
  • To identify high-traffic areas and pain points (KPIs)
  • To identify potential usability issues at scale
  • Before and after launching new features
  • To understand user paths and drop-off points
  • For identifying user behavior patterns

When not to use

  • When you need qualitative insights about user motivations
  • For gathering contextual information
  • When data sample size is too small
  • For testing new concepts without historical data
  • When tracking isn't properly implemented

Process

1

Preparation

  • Define analysis objectives and hypotheses
  • Verify tracking implementation accuracy
  • Identify key metrics and KPIs
  • Set up data collection filters and segments
2

Data Quality Check

  • Extract relevant data sets
  • Remove outliers and anomalies
  • Standardize data format
  • Verify data integrity across sources
  • Address missing data points
3

Analysis & Pattern Recognition

  • Identify trends and patterns
  • Map user journeys and flows
  • Analyze conversion funnels
  • Compare data across metrics
  • Segment user behaviors
  • Discover behavioral cohorts
4

Synthesis & Communication

  • Create visual representation of findings
  • Document methodology and limitations
  • Develop actionable insights
  • Prioritize opportunities
  • Propose substantiate recommendations

Example Questions

  • "What are the typical user journeys through the product?"
  • "Where do users most frequently abandon their journey?"
  • "Which features are most/least used?"
  • "How do different user segments behave?"
  • "What is the impact of recent changes on key metrics?"

Pairs well with

A/B Testing

Why?

Analytics review helps identify opportunities for testing and provides baseline metrics for measuring A/B test effectiveness.

User Interviews

Why?

Combine quantitative analytics data with qualitative interviews to understand both what users do and why they do it.
