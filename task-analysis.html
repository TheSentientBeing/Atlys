Discovery Method

Task Analysis

Detailed examination of how users perform tasks, identifying steps, requirements, and mental models for understanding user activities.

Time Required

2-3 Weeks

Participants Needed

5-8 Users

Difficulty Level

Advanced

Overview

Task analysis systematically breaks down how users complete specific goals and activities. This method reveals the steps, decisions, tools, and knowledge required to complete tasks, helping teams understand user workflows and mental models at scale.

Usage

When to use

  • Understanding complex workflows
  • Designing new processes
  • Improving existing flows
  • Training development
  • Identifying requirements
  • Mapping decision points
  • Documenting procedures
  • Optimising efficiency

When not to use

  • For simple, obvious tasks
  • When gathering opinions
  • Testing visual design
  • Measuring satisfaction
  • For broad solutions
  • When time is limited
  • Without clear task focus

Process

1

Preparation

  • Select tasks to analyse
  • Define scope
  • Identify user groups
  • Choose analysis methods
  • Create observation plan
  • Prepare documentation tools
2

Data Collection

  • Observe task performance
  • Document steps
  • Record decisions
  • Note tools used
  • Track time spent
  • Identify problems
  • Gather contextual information
3

Task Breakdown

  • List individual steps
  • Map decision points
  • Document triggers
  • Identify dependencies
  • Note required knowledge
  • Document variations
  • Map task relationships
4

Requirement Analysis

  • List tools needed
  • Document knowledge required
  • Identify skills needed
  • Note environmental factors
  • Map information flows
  • List assumptions
5

Pattern Recognition

  • Identify common paths
  • Note variations
  • Find inefficiencies
  • Spot workarounds
  • Map mental models
  • Document pain points
6

Synthesis & Documentation

  • Create task flows
  • Write procedure guides
  • Note requirements
  • Map decision trees
  • Make recommendations
  • Present findings

Example Questions

  • "What triggers this task?"
  • "What information is needed?"
  • "What decisions are made when?"
  • "Which tools are required?"
  • "How do different users approach this?"

Pairs well with

User Interviews

Why?

Combine task analysis with interviews to understand underlying goals and mental models behind observed behaviours.

Journey Mapping

Why?

Use task analysis to provide detailed understanding of specific activities within the broader user journey.
