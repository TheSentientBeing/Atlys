Evaluation Method

Prototype Testing

Evaluate interactive prototypes to validate design concepts and interactions before investing in development. Identifies usability issues early.

Time Required

1-3 Weeks

Participants Needed

5-8 Users

Difficulty Level

Intermediate

Overview

Prototype testing involves observing users as they interact with preliminary versions of a design. This method helps validate assumptions, interaction patterns, and usability early in the design process when changes are less costly and easier to implement.

Usage

When to use

  • Validating design concepts
  • Testing interaction patterns
  • Evaluating navigation flows
  • Before development begins
  • During iterative design
  • Comparing design alternatives
  • Gathering early feedback

When not to use

  • For final product validation
  • When testing performance
  • For long-term usage patterns
  • Testing backend functionality
  • When gathering metrics
  • For production releases
  • Testing complex systems

Process

1

Preparation

  • Define test objectives
  • Create test scenarios
  • Build prototype
  • Write test script
  • Set success criteria
  • Prepare testing environment
2

Participant Recruitment

  • Define participant profiles
  • Screen candidates
  • Schedule sessions
  • Send instructions
  • Arrange incentives
3

Testing Sessions

  • Welcome participants
  • Explain process
  • Present tasks
  • Observe interactions
  • Take notes
  • Record sessions
  • Gather feedback
4

Analysis

  • Review recordings
  • Compile observations
  • Identify patterns
  • Map pain points
  • Note suggestions
  • Assess task success
5

Iterations

  • Prioritize issues
  • Update prototype
  • Document changes
  • Plan next round
  • Share learnings

Example Questions

  • "What are you thinking as you look at this?"
  • "What would you expect to happen next?"
  • "How would you complete this task?"
  • "What confused you about this interaction?"
  • "How does this compare to what you currently use?"

Pairs well with

User Interviews

Why?

Continue prototype testing with in-depth interviews to understand users' mental models and expectations.

First Click Testing

Why?

Use first click testing to validate specific interaction points within the prototype.
